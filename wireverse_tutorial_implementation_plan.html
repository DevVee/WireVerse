1
Orientation — understand before doing
5–7 min ›
Goal: Player knows what they're building, why it matters, and what each part does before touching anything.
Implementation tasks
UI
Scene: Workshop / Classroom mode
Dedicated non-interactive environment. NPC or narrator voice sets context: "Today we wire a basic lighting circuit." No draggable components yet.
IC
Interactive component inspector
Click any component to open a pop-up: Name, Symbol (TESDA standard), Function. Example — click bulb → "Converts electrical energy to light. Symbol: ⊙". Must include TESDA schematic symbols to support NC assessments.
SF
Safety briefing overlay
Before any wiring: display 2–3 safety rules (de-energize before wiring, avoid short circuits). Require explicit acknowledgement. Log compliance for reporting. must-have
SC
Split-screen schematic preview
Left: TESDA diagram. Right: 3D/isometric view of the same circuit. Player sees the connection between symbol and physical object before building. nice-to-have v1.1
2
Demonstration — watch the system build it
2–3 min ›
Goal: Reduce cognitive load. Player sees the complete solution first, then attempts it themselves.
AN
Animated step-by-step auto-build
System draws each wire segment with a slow animated stroke (200–400ms per segment). Highlight the active wire in accent color. Auto-pause between steps so player can read the caption.
CA
Step captions with narration
Each animation step shows text: "Connect live terminal of source to switch input." Voice-over optional but recommended for TESDA training context. Subtitle always visible.
CT
Playback controls
Pause, rewind one step, replay full demo. Player can re-watch any step. This directly reduces frustration in slower learners — important for NC II audience.
HL
Terminal highlights
During demo, source terminal glows before wire connects to it. Makes "which terminal" obvious before the player has to figure it out alone in Phase 3.
3
Guided Practice — do it with support
5–10 min ›
Goal: Player performs each step with safety rails. Mistakes are corrected with hints, not failure screens.
SN
Snap system with valid-only nodes
During guided mode, only the correct next terminal is selectable. Wire snaps to target when within ~20px. Prevents frustration from missed clicks, keeps focus on learning not fine motor skill.
GL
Glow hints on correct nodes
Correct terminal pulses softly when player is expected to connect it. Hint fades after 3 seconds; reappears if player hasn't acted. No text required — the glow is the cue.
FB
Context-specific wrong-attempt feedback
Wrong node: show red flash + specific message: "Check the switch terminals — live goes to L1." Not generic "incorrect." Message references the component name the player just tried. must-have
ST
Step counter + progress breadcrumb
Show "Step 2 of 5: Connect switch to bulb." Player always knows how much is left and what they're doing now. Reduces anxiety in assessment-minded learners.
4
Independent Task — assess real learning
5–8 min ›
Goal: No hints, no glows, no snapping. Player builds from memory. This is the assessment phase.
AS
Full free-build mode
All terminals selectable. No glow cues. Wrong connections are accepted but logged for the post-task report — do NOT interrupt with error messages mid-build. Let player finish, then analyze.
TM
Timer + attempt counter
Visible timer. Track number of incorrect wires placed. Both feed into the post-task score. Timer is visible but not stressed — this is a learning tool, not a speed test.
PW
Power ON test
Player clicks "Power ON" when done. If circuit is correct: bulb lights up with animation + sound. If incorrect: identify the fault (e.g., "Open circuit detected at Switch L1"). Outcome drives the feedback report.
SP
Safety violation detection
If player wires a short circuit or bypasses switch: flag as safety violation in report. Deduct points. In future: animate a "fault" spark effect as consequence. must-have
Feedback system
●
Real-time wire feedback
Correct → green wire flash + chime. Wrong → red glow + specific message referencing the component (not "incorrect"). Guided mode only.
●
Post-task performance report
After Power ON: Accuracy %, Mistakes count, Time taken, list of specific errors ("Confused live and neutral on bulb"). Exportable for TESDA record-keeping. must-have
●
Reinforcement routing
On fail: offer "Retry guided mode" or "Review component lesson." Never just "Try again." Route to specific weak point, not the start of the module.
●
Safety incident logging
Short circuits, skipped procedures: logged separately from accuracy score. Shown prominently in report. Mirrors real TESDA NC II safety competency requirement. must-have
Engagement + gamification
Stars / Rating
0–3 stars per module based on accuracy + safety score. Never block progress on stars — they're a reward, not a gate.
Unlock system
Complete Basic Light Circuit → unlock 2-way switch module. Tools unlock progressively (e.g., multimeter unlocks after Phase 4 pass).
Progress bar
Module map shows phases as nodes. Current phase highlighted. Completed phases show star rating. At-a-glance curriculum view.
Personal best
Track best time + best accuracy per module. Player can replay any module to beat their record. Encourages mastery without pressure.
Educational differentiators (thesis panel highlights)
●
Schematic ↔ 3D split screen
Phase 1 + 2: left panel shows TESDA diagram, right shows 3D view. Highlighted components sync between panels. Builds mental mapping skill critical for NC II practical exam.
●
Mastery-based progression
Player cannot advance to the next module until Phase 4 is passed with ≥80% accuracy AND zero safety violations. Mirrors TESDA competency-based training (CBT) philosophy.
Recommended build order for "Basic Light Circuit" module. Ship each step before starting the next.
1
Component inspector (Phase 1 core)
Static scene + clickable components with name, symbol, function. No wiring yet. This is your demo-able proof of concept for a panel presentation.
2
Safety briefing screen
Modal with 3 rules, acknowledgement button. Simple but critical — establishes safety-first culture and satisfies TESDA competency framing.
3
Animated demo (Phase 2 core)
Auto-builds circuit with step captions. No player input. Focus on smooth animation and readable captions. Pause/replay buttons.
4
Wire drag + snap mechanics
Core interaction engine. Terminal hitboxes, wire rendering, snap radius. Build this once and reuse for both Phase 3 and Phase 4. Most technically complex step.
5
Guided mode layer (Phase 3)
Add glow hints, valid-node filtering, and context-specific feedback messages on top of Step 4. Keep Phase 4 (assessment) toggled off during this step.
6
Power ON test + bulb animation
Circuit validation logic + success/fault visual. Bulb glow on success, fault message on failure. This is your "wow moment" — make it feel satisfying.
7
Post-task report screen
Accuracy, time, mistake list, safety flag. Simple layout. This closes the feedback loop and makes the tool feel genuinely educational, not just a game.
8
Full Phase 4 (assessment mode)
Strip hints, enable all terminals, add timer. Routes to report from Step 7. With this done, the full tutorial loop is complete and thesis-defensible.
